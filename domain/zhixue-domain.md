# 智学网领域知识

> 本文件为 write-prd 技能提供智学网业务上下文。AI 在生成 PRD 时参考此文件，
> 确保用户角色、数据表、指标定义等术语准确一致。

## 实体关系

```
学校(School)
├── 年级(Grade)
│   └── 班级(Class)
│       └── 学生(Student)
└── 教师(Teacher)
    ├── 任课教师（绑定学科）
    ├── 班主任（绑定班级，可同时是任课教师）
    ├── 教研组长（绑定学科，跨班职责）
    └── 教务主任 / 校长（校级管理）

考试(Assessment) ── 关联 → 班级 + 学科
├── 类型：单元测 / 月考 / 期中 / 期末 / 模拟考
├── 注意：不同类型满分不同，不能直接比较绝对分数
└── 题目 → 关联知识点(Knowledge Point)

错题本(Wrong Answer Note) ── 关联 → 学生 + 题目 + 知识点
└── 状态：待复习 / 已复习 / 已掌握
```

## 核心数据表

PRD 中"数据依赖"章节常用的表和字段：

| 表名 | 说明 | 常用字段 | PRD 中的典型用途 |
|------|------|---------|----------------|
| assessment | 考试成绩 | student_id, score, exam_type, subject, question_type | 成绩分析、分数段分布、进退步 |
| event_log | 用户行为 | user_id, event_type, timestamp, page | 使用率、停留时间、功能渗透率 |
| knowledge_point | 知识点 | subject, chapter, point_name, mastery_rate | 薄弱知识点、掌握度热力图 |
| contract | 学校合同 | school_id, start_date, end_date, amount, status | 续费率、ARPU |
| teacher | 教师信息 | teacher_id, role, subject, school_id | 角色自适应、学科模板匹配 |
| school | 学校信息 | school_id, province, city, school_type, size | 区域分析、灰度分组 |
| wrong_note | 错题本记录 | student_id, question_id, knowledge_point_id, status | 错题本使用率、学习行为 |

## 指标速查

PRD 中"成功指标"章节常用的指标精确定义：

### North Star

**周活跃学生完成考试数** = 该自然周内至少完成 1 次正式考试的去重学生数

### L1 健康指标

| 指标 | 精确定义 | 数据源 |
|------|---------|--------|
| 新签约学校数 | 当月签署合同的学校数（含续约转新合同） | contract 表 |
| 新校激活率 | 签约 30 天内有学生使用错题本的学校 / 当月新签约学校 | event_log + contract |
| 学校续费率 | 到期续约学校 / 当期到期学校（按自然年，排除测试学校） | contract 表 |
| 教师 NPS | 季度调研中回答"愿意推荐"的教师比例 | 问卷系统 |
| ARPU | 年度合同总收入 / 活跃学校数 | 财务 + 产品 |

### L2 诊断指标

| 指标 | 精确定义 | 常见用途 |
|------|---------|---------|
| 教师报表使用率 | 月内至少查看 1 次学情报告的教师 / 注册教师（排除已离职） | 诊断教师满意度 |
| 错题本使用率 | 月内至少使用 1 次错题本的学生 / 月活跃学生 | 诊断留存 |
| 功能渗透率 | 月内使用某功能的学校 / 活跃学校 | 诊断产品健康度 |

### 数据陷阱提醒

在 PRD 中设定指标时需注意：
- 分母要明确（活跃用户 vs 注册用户 vs 全部用户）
- 考试周期有季节性（考试月数据天然高，环比无意义，应看同比）
- 续费率只在到期时点统计，不用全部学校做分母
- 不同考试满分不同，直接算平均分无意义

## 用户角色术语

PRD 中描述用户时，必须使用以下角色术语（不要泛化为"用户"）：

| 角色 | 关注点 | PRD 中的典型场景 |
|------|--------|----------------|
| 学生 | 成绩、错题、知识点掌握 | 学习功能、错题本、成绩查询 |
| 任课教师 | 班级学情、学科诊断、教学调整 | 报表、知识点分析、作业布置 |
| 班主任 | 全班各科总览、关注学生 | 跨学科对比、退步预警 |
| 教研组长 | 同学科跨班对比、教师支持 | 年级学科报告、教学质量 |
| 校长/教务主任 | 全校数据、办学效果 | 校级看板、对外汇报 |
| 教育局 | 区域覆盖、教学质量均衡 | 区域看板、政策对齐 |
| 学校管理员 | 账号管理、数据导入 | 后台管理、考试设置 |
| 家长 | 孩子成绩、学习建议 | 家长端推送、成绩单 |

## 团队角色

PRD 中"负责人"和"确认人"常用的团队角色：

| 角色 | 在 PRD 中的典型职责 |
|------|-------------------|
| 产品 | PRD 撰写、需求对齐、优先级决策、灰度方案 |
| 设计 | 交互稿、视觉稿、可用性测试 |
| 前端工程 | 页面开发、模板渲染、性能优化 |
| 后端工程 | 数据接口、预计算、字段扩展、性能优化 |
| 教研团队 | 学科指标确认、知识点映射、题型分类 |
| 运营 | 灰度推送、教师通知、学校沟通 |
| 数据/BI | 指标埋点、监控看板、数据验证 |
| 测试/QA | 功能测试、性能测试、验收 |

## 技术上下文

- 教师端：Web（PC 为主，移动端为辅）
- 学生端：App + Web
- 管理端：Web
- 数据报表：后端预计算 + 前端渲染
- 行为埋点：event_log 表（event_type 字段区分）
- 排除规则：测试学校 `school_type = 'test'`、内部员工账号
