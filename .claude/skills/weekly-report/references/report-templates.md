# 周报模板与写作指引

## 状态信号定义

### Green / Yellow / Red 判定标准

| 状态      | 含义                   | 智学网场景示例                                |
| --------- | ---------------------- | --------------------------------------------- |
| 🟢 Green  | 按计划推进，无重大风险 | 需求按时交付设计评审，指标稳定                |
| 🟡 Yellow | 有风险但可控，需关注   | 考试季排期压缩、某指标连续 2 周微降、竞品动态 |
| 🔴 Red    | 偏离计划，需干预       | 关键需求延期超 1 周、续费率下滑、线上事故     |

**判定原则**：

- 主动标 🟡 是好的风险管理，不是示弱
- 只在真正需要帮助时标 🔴
- 状态变化时注明原因："上周 🟢 → 本周 🟡，因为 [原因]"

### 风险分级（ROAM）

| 分级      | 含义   | 周报写法                                  |
| --------- | ------ | ----------------------------------------- |
| Resolved  | 已解决 | ✅ [风险]：已通过 [方案] 解决             |
| Owned     | 已认领 | 🔧 [风险]：[负责人] 正在处理，预计 [时间] |
| Accepted  | 已接受 | ℹ️ [风险]：已知但选择不处理，因为 [原因]  |
| Mitigated | 已缓解 | ⬇️ [风险]：已通过 [措施] 降低影响         |

## 领导版模板

```markdown
# 产品周报 — {日期}

**状态**：🟢 / 🟡 / 🔴
**一句话**：[本周最重要的一件事]

## 本周成果

1. [成果]（影响：[指标变化/里程碑达成]）
2. [成果]（影响：[xxx]）
3. [成果]（影响：[xxx]）

## 指标速览

| 指标    | 本周 | 上周 | 变化     | 状态     |
| ------- | ---- | ---- | -------- | -------- |
| [指标1] | [值] | [值] | [↑/↓ X%] | 🟢/🟡/🔴 |
| [指标2] | [值] | [值] | [↑/↓ X%] | 🟢/🟡/🔴 |

## 风险

- [ROAM 分级] [风险描述]

## 需要决策

- [决策事项]：[选项 + 建议]。需在 [日期] 前确定。

## 下周重点

1. [重点1]
2. [重点2]
3. [重点3]
```

**写作指引**：

- 总字数控制在 200 字以内
- 成果要绑定指标或里程碑，不写"开了 X 个会"
- 风险只列需要领导关注的，自己能解决的不列
- 决策需求要给建议方案，不要只抛问题

## 研发版模板

```markdown
# 产品→研发 周同步 — {日期}

## 需求变更

| 需求     | 变更内容   | 原因   | 影响范围            |
| -------- | ---------- | ------ | ------------------- |
| [需求名] | [变更说明] | [原因] | [影响哪些模块/接口] |

## 优先级调整

- [需求] P1→P0：[原因]
- [需求] 新增 P1：[背景]

## 阻塞项

| 阻塞       | 等待谁    | 预计解除 | PM 在做什么 |
| ---------- | --------- | -------- | ----------- |
| [阻塞说明] | [人/团队] | [日期]   | [推动动作]  |

## 本周 Sprint 状态

- ✅ 已完成：[需求列表]
- 🔄 进行中：[需求列表]
- ⏸ 延期：[需求 + 原因]

## 下周排期

- [即将进入开发的需求]
- [设计评审通过待排期的需求]
```

**写作指引**：

- 需求变更说明"为什么变"，研发更容易配合
- 阻塞项写清楚"PM 在做什么推动"，让研发知道你在解决
- 优先级调整要有原因，不能只说"领导要求"

## 学校版模板

```markdown
# 智学网产品动态 — {日期}

## 新功能上线

### [功能名称]

[一句话说明这个功能帮学校/老师解决什么问题]

**怎么用**：[操作路径，如：登录 → 报表 → 点击"精简视图"]

## 使用建议

- [建议1：帮助学校更好使用产品的提示]
- [建议2]

## 已知问题

| 问题       | 影响范围              | 预计修复 |
| ---------- | --------------------- | -------- |
| [问题描述] | [哪些学校/功能受影响] | [日期]   |

## 反馈

如有使用问题或建议，请联系：[联系方式]
```

**写作指引**：

- 不用内部术语（Sprint、P0、OKR、backlog）
- 功能用"帮你解决什么问题"描述，不用"我们做了什么"
- 已知问题只列影响用户的，内部 bug 不列
- 语气亲和，像在跟老师聊天

## 智学网特有节奏

| 时间                 | 事件     | 对周报的影响                       |
| -------------------- | -------- | ---------------------------------- |
| 每年 3 月、9 月      | 开学季   | 学校版重点：新功能引导、开学配置   |
| 期中考试前 2 周      | 考试准备 | 研发版：冻结非紧急需求             |
| 期末考试月           | 高峰期   | 领导版：关注系统稳定性和并发指标   |
| 暑假/寒假            | 低峰期   | 适合推大版本更新、数据迁移         |
| 续费季（通常 Q3-Q4） | 续费评估 | 领导版：强调续费相关指标和客户反馈 |
