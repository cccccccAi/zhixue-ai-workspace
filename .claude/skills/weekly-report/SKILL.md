# 智学网干系人周报生成器

## 执行协议

- 产品同行语气，不教基础
- **全部中文输出**
- 每次输出 ≤ 15 行（不含代码块和表格），超过时用 AskUserQuestion 过渡
- 产出文件命名：`weekly-report-{日期}.md`，存入 `./learning/outputs/`
- md 文件生成后用 `open -a "MarkText"` 自动打开
- 不确定的内容用 `[待填: 描述]` 占位

---

## 启动流程

### 1. 解析输入

解析 `$ARGUMENTS`：

- **有内容** → 作为本周进展概要，跳到 Step 2
- **空** → 进入 Step 1

### 2. 检测历史周报

自动检测 `./learning/outputs/weekly-report-*.md`：

- **存在** → 读取最近一份，作为"上周状态"参考
- **不存在** → 首次生成，跳过对比

---

## 三步工作流

### Step 1：收集进展

用 AskUserQuestion 问（每个单独问）：

**第一个**：本周成果

```
本周做了哪些事？（可以是碎片化的，我帮你整理）
- 粘贴文字/笔记
- 口述几个关键词
- 从飞书/钉钉复制消息
```

**第二个**：风险和阻塞

```
有没有阻塞或风险？
- 有，我说一下
- 没有，一切顺利
- 有一些但不严重
```

### Step 2：选择读者

用 AskUserQuestion 问：

```
这份周报给谁看？
- 领导层（关注成果、指标、风险）
- 研发团队（关注需求变更、优先级、阻塞）
- 学校/客户（关注新功能、使用指南）
- 多个版本都要（推荐）
```

### Step 3：生成周报

根据读者生成对应版本，参考 [report-templates.md](references/report-templates.md)。

**领导版** — 简洁、数据驱动：

```
状态：🟢 / 🟡 / 🔴
一句话总结：[最重要的一件事]
本周成果：[3-5 条，每条绑定指标或里程碑]
风险：[ROAM 分级 — 已解决/已认领/已接受/已缓解]
决策需求：[需要领导拍板的事]
下周重点：[Top 3]
```

**研发版** — 具体、可执行：

```
需求变更：[变更清单 + 原因]
优先级调整：[P0/P1 变动]
阻塞项：[谁在等什么，预计何时解除]
Sprint 状态：[进行中 / 已完成 / 延期]
下周排期：[即将进入开发的需求]
```

**学校版** — 简单、友好：

```
新功能上线：[功能名 + 一句话说明 + 操作入口]
使用建议：[帮助学校/教师用好的提示]
已知问题：[有的话说明 + 解决时间]
反馈渠道：[联系方式]
```

**生成后**：

1. 用 Write 写入 `./learning/outputs/weekly-report-{日期}.md`
2. 用 `open -a "MarkText"` 打开
3. 输出简要确认：生成了几个版本，核心内容概要

## 状态判定规则

参考 [report-templates.md](references/report-templates.md) 中的 Green/Yellow/Red 定义。

基本原则：

- **🟢 Green**：按计划推进，无重大风险
- **🟡 Yellow**：有风险但可控，需要关注（主动标黄是好的风险管理）
- **🔴 Red**：严重偏离计划，需要干预或资源支持

智学网特有判定：

- 考试季（期中/期末）阻塞了需求排期 → 🟡（季节性，非产品问题）
- 学校续费率连续下降 → 🔴
- 竞品上线了差异化功能 → 🟡

---

## Anti-Patterns

- **不要编造进展**：用户没说的不要脑补
- **不要写流水账**：按成果写，不按时间线写
- **领导版不要超过 200 字**：简洁是核心价值
- **学校版不要用内部术语**：不写"Sprint""P0""OKR"
- **不要跳过风险**：没有风险也要说"本周无重大风险"
