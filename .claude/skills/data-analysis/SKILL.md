# 智学网产品数据分析

## 执行协议

- 产品同行语气，不教基础
- **全部中文输出**
- 每次输出 ≤ 15 行（不含代码块），超过时用 AskUserQuestion 过渡
- 产出文件命名：`data-analysis-{主题}-{日期}.md`，存入 `./learning/outputs/`
- 生成后用 `open -a "MarkText"` 自动打开
- 不确定的数据用模拟数据，但在报告中标注"模拟数据"
- **先说结论，再给过程**——分析报告的每一节都以一句话结论开头

---

## 启动流程

### 1. 解析输入

解析 `$ARGUMENTS`：

- **有具体问题**（如"为什么活跃率下降"）→ 直接进入分析流程
- **有数据描述**（如"分析这份考试数据"）→ 询问数据来源
- **空** → 用 AskUserQuestion 选择分析场景

### 2. 加载领域知识

读取 [zhixue-domain.md](../../domain/zhixue-domain.md) 获取实体、指标和角色定义。

---

## 四步工作流

### Step 1：明确分析目标

用 AskUserQuestion 问（如果 $ARGUMENTS 不够明确）：

**第一个**：分析场景

```
你想分析什么？
- 趋势分析（某指标的变化趋势，找拐点和规律）
- 对比分析（不同维度的对比，如年级vs年级、学科vs学科）
- 归因分析（某指标异常，找原因）
- 分布分析（数据分布特征，如分数段、使用频次）
- 我直接描述问题
```

**第二个**：分析深度

```
分析到什么程度？
- 快速回答（1-2 个关键结论，适合口头汇报）
- 标准分析（完整分析报告，适合邮件/飞书分享）（推荐）
- 深度报告（含多维交叉分析和行动建议，适合专题会议）
```

### Step 2：数据输入

用 AskUserQuestion 问：

```
数据从哪来？
- 我粘贴数据（CSV/表格/文字描述均可）
- 用模拟数据演示分析方法（推荐）
- 我描述数据结构，帮我搭分析框架
```

**如果用户粘贴数据**：

- 解析数据格式，确认字段含义
- 检查数据质量（缺失值、异常值、时间范围）
- 提示数据量级和覆盖范围

**如果用模拟数据**：

- 按分析场景生成合理的模拟数据
- 在报告中标注"⚠️ 以下分析基于模拟数据，仅演示分析方法"

### Step 3：执行分析

按分析类型选择方法论（参考 [analysis-methods.md](references/analysis-methods.md)）：

**趋势分析**：

- 时间序列拆解：趋势 + 周期 + 异常
- 环比/同比计算
- 拐点识别和可能原因

**对比分析**：

- 多维度交叉对比
- 基准线设定（全校均值、行业基准）
- 差异显著性判断（大幅偏离 vs 正常波动）

**归因分析**：

- 构建假设树（产品因素 / 外部因素 / 数据因素）
- 逐一验证或排除
- 按可能性排序

**分布分析**：

- 描述性统计（均值、中位数、标准差、分位数）
- 分段特征（各分数段/频次段的人数和占比）
- 异常值识别

**分析过程中自动执行验证清单**（参考 [validation-checklist.md](references/validation-checklist.md)）：

- 数据质量 → 计算逻辑 → 合理性 → 呈现准确性

### Step 4：输出分析报告

**报告结构**（标准分析）：

```markdown
# [分析主题] — 数据分析报告

> 数据截止：{日期} | 分析类型：{类型} | 数据来源：{来源}

## 核心结论（1-3 句话）

[最重要的发现，直接回答分析问题]

## 数据概览

[数据范围、样本量、时间跨度、关键字段]

## 分析过程

### [发现 1 标题]

[结论先行 → 数据支撑 → 解读]

### [发现 2 标题]

[结论先行 → 数据支撑 → 解读]

## 建议行动

| 优先级 | 行动 | 依据 | 负责方向 |
| ------ | ---- | ---- | -------- |

## 数据验证

- [x] 数据完整性检查
- [x] 计算逻辑验证
- [x] 结论合理性校验

## 局限与说明

[数据局限、分析假设、需要进一步验证的点]
```

**快速回答**：只输出核心结论 + 关键数据支撑，不生成文件

**深度报告**：在标准分析基础上增加：

- 多维交叉分析表
- 历史趋势对比
- 竞品/行业基准参照
- 详细行动计划（含时间线）

**输出**：

1. 用 Write 写入 `./learning/outputs/data-analysis-{主题}-{日期}.md`
2. 用 `open -a "MarkText"` 打开
3. 简要确认：

```
分析报告已生成：
- 分析类型：[趋势/对比/归因/分布]
- 核心发现：[一句话]
- 建议行动：[X] 条
- 数据验证：已通过

文件：learning/outputs/data-analysis-{主题}-{日期}.md
```

---

## Anti-Patterns

- **不要只列数据不给结论**：每个数据点都要回答"所以呢？"
- **不要忽视数据验证**：分析结果必须过验证清单
- **不要混淆相关和因果**：归因分析中明确标注"相关"还是"可能因果"
- **不要忽略季节性**：教育数据受学期节奏影响大（考试月、寒暑假）
- **不要省略局限说明**：模拟数据必须标注，真实数据也要说明覆盖范围
