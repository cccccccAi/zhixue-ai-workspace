# 智学网数据看板生成器

## 执行协议

- 产品同行语气，不教基础
- **全部中文输出**，看板界面也用中文
- 每次输出 ≤ 15 行（不含代码块），超过时用 AskUserQuestion 过渡
- 产出文件命名：`dashboard-{名称}-{日期}.html`，存入 `./learning/outputs/`
- HTML 文件生成后用 `open -a "Google Chrome"` 自动打开
- 不确定的数据用模拟数据，但在看板上标注"模拟数据"

---

## 启动流程

### 1. 解析输入

解析 `$ARGUMENTS`：

- **含"教师"** → 使用教师看板预设
- **含"学校"或"管理"** → 使用学校管理看板预设
- **含"产品"** → 使用产品团队看板预设
- **有其他内容** → 作为自定义看板描述
- **空** → 用 AskUserQuestion 选择看板类型

### 2. 加载领域知识

读取 [zhixue-domain.md](../../domain/zhixue-domain.md) 获取实体、指标和角色定义。

## 四步工作流

### Step 1：看板定义

用 AskUserQuestion 问（每个单独问）：

**第一个**：看板受众

```
这个看板给谁看？
- 教师（班级考试情况、学生表现、知识点分析）
- 学校管理员（年级/学科概览、教师使用率、考试频次）
- 区域教育局（学校排名、区域均分、使用渗透率）
- 产品团队（DAU/MAU、功能使用率、续费率、性能）
- 自定义（我描述需求）
```

**第二个**：核心指标

```
看板重点展示什么？
- 用预设指标（推荐，按受众自动配置）
- 我指定要看的指标
```

### Step 2：数据输入

用 AskUserQuestion 问：

```
数据从哪来？
- 我粘贴数据（CSV/表格/JSON）
- 用模拟数据先看效果（推荐）
- 描述数据结构，帮我生成模拟数据
```

**如果用户粘贴数据**：

- 解析数据格式，映射到看板组件
- 数据量大时提示：只取最近 N 条

**如果用模拟数据**：

- 按受众预设生成合理的模拟数据
- 在看板顶部标注"⚠️ 模拟数据，仅供演示"

### Step 3：生成看板

基于 [dashboard-components.md](references/dashboard-components.md) 中的组件库生成。

**HTML 结构**（单文件，自包含）：

```
<!DOCTYPE html>
├── Chart.js CDN 引用
├── 内联 CSS（响应式布局）
├── 看板标题 + 日期 + 数据标注
├── 筛选器行（下拉菜单，按需）
├── KPI 卡片行（3-5 个核心指标）
├── 图表区域（2-4 个图表）
├── 数据表格（可排序）
├── 页脚（数据截止时间）
└── 内嵌 JSON 数据 + Dashboard 类
```

**按受众配置**（参考 [dashboard-presets.md](references/dashboard-presets.md)）：

- 教师看板：均分+排名+薄弱知识点 Top3+分数段分布+趋势对比
- 学校看板：各年级均分+教师使用率+考试频次+环比变化
- 产品看板：DAU/MAU+功能使用率+续费率+加载性能

**技术要求**：

- Chart.js 4.x（CDN 引用，含 SRI hash）
- 所有数据内嵌为 JSON（不依赖外部文件）
- 筛选器实时更新所有图表
- 配色：智学网品牌蓝 #1890FF 为主色
- 支持打印（@media print 隐藏筛选器，保留图表）

### Step 4：输出

1. 用 Write 写入 `./learning/outputs/dashboard-{名称}-{日期}.html`
2. 用 `open -a "Google Chrome"` 打开预览
3. 输出简要确认：

```
看板已生成，包含：
- KPI 卡片：X 个
- 图表：X 个（柱状图/折线图/...）
- 筛选器：X 个
- 数据：[真实/模拟]

文件：learning/outputs/dashboard-{名称}-{日期}.html
可以直接发给 [受众] 查看，无需安装任何工具。
```

## Anti-Patterns

- **不要生成需要服务器的看板**：必须是单文件 HTML，浏览器直接打开
- **不要用太多图表**：一个看板 2-4 个图表足够，多了反而看不过来
- **不要混淆受众**：教师不需要看 DAU，产品不需要看知识点
- **不要省略模拟数据标注**：用模拟数据必须在看板上标注
- **不要忘记打印支持**：学校管理员经常需要打印看板
