# 真实数据填充交互指南

> 本文件定义 PRD 生成后的数据填充交互流程。AI 按此脚本引导 PM
> 逐步替换占位符为真实数据。

## 占位符类型

PRD 中会出现以下占位符：

| 占位符格式 | 含义 | 示例 |
|-----------|------|------|
| `[待填: 描述]` | PM 需要补充的真实数据 | `[待填: 报表使用率当前值]` |
| `[待确认: 描述]` | 需要跨团队确认的假设 | `[待确认: teacher.role 字段覆盖率]` |
| `[估算值: 数值，基于X推算]` | PM 选择估算后 AI 辅助生成 | `[估算值: ~65%，基于去年同期+增长率推算]` |

## 扫描规则

AI 读取生成的 PRD 文件，用正则搜索所有占位符：
- 搜索 `\[待填:` `\[待确认:` `\[估算值:`
- 按所在章节归类
- 按重要性排序：先行指标的当前值 > 目标值 > 数据依赖状态 > 负责人/时间

## 交互流程

### 第一步：展示数据缺口概览

```
━━━━━━━━━━━━━━━━━━━━━━
  数据缺口清单
━━━━━━━━━━━━━━━━━━━━━━

  必填项：X 项
  ├ 指标当前值：X 项
  ├ 指标目标值：X 项
  └ 负责人/截止时间：X 项

  待确认项：X 项

  总计：X 项
━━━━━━━━━━━━━━━━━━━━━━
```

然后用 AskUserQuestion 询问：

```
选项：
- "逐项填充"（推荐，AI 分批引导）
- "我先看看 PRD 全文，一会儿自己改"
- "跳过数据填充，直接质检"
```

### 第二步：分批填充

每批处理 3-4 个同类占位符。展示一个简表后用 AskUserQuestion 提问。

**指标当前值批次示例**：

```
以下指标需要填入当前真实值：

| 章节 | 指标 | 占位符 |
|------|------|--------|
| 第10节 | 教师报表使用率 | [待填: 当前值] |
| 第10节 | 学校续费率 | [待填: 当前值] |
| 第1节 | 使用率下降幅度 | [待填: 起始值→当前值] |
```

AskUserQuestion 选项：
- "我知道这些数据，告诉你"
- "我需要查一下后台，先跳过这批"
- "部分知道部分不知道"

**应答处理**：

- **"我知道"**：PM 粘贴/口述数据。AI 直接替换 PRD 中的占位符，用 Edit 工具更新文件。
- **"需要查"**：保留占位符，改为 `[待填: 教师报表使用率当前值 — 评审前补充]`
- **"部分知道"**：AI 逐个询问，知道的替换，不知道的保留

### 第三步：估算辅助

如果 PM 选择"改为估算"，AI 提供估算辅助：

```
估算思路：
- 上一个已知值是多少？
- 是上升趋势还是下降趋势？
- 有没有可参考的行业基准？

基于你提供的信息，建议估算值为 [X]。
在 PRD 中标注为：
[估算值: X，基于Y推算，需Z确认]
```

### 第四步：跨团队确认项

扫描所有 `[待确认:]` 占位符，生成确认清单追加到 PRD 末尾：

```markdown
## 附录 B：跨团队确认清单

> 以下事项需要在设计/开发启动前与对应团队确认。
> 建议将此表复制发送给对应负责人。

| # | 确认项 | 问谁 | 预期回答格式 | 截止 | 状态 |
|---|--------|------|------------|------|------|
| 1 | teacher.role 字段覆盖率 | 后端工程 | 百分比（如"85% 已标记"） | [待填: 日期] | 未确认 |
| 2 | 语文题型标签是否齐全 | 教研团队 | 是/否 + 缺失列表 | [待填: 日期] | 未确认 |
```

### 填充完成

所有批次处理完毕后：
- 用 Edit 工具把所有变更写入 PRD 文件
- 在 MarkText 中重新打开更新后的文件
- 输出填充统计：

```
数据填充完成：
  ✅ 已填充：X 项
  ⏳ 待补充：X 项（已标记在 PRD 中）
  📋 待确认：X 项（已生成确认清单）
```

然后自动进入 Step 5 交付质检。

## Anti-Patterns

- 不要替 PM 编造看似真实的数字
- 不要一次问超过 4 个数据项（信息过载）
- 不要把"待确认"项当作"已确认"处理
- PM 说"跳过"就跳过，不要反复追问
